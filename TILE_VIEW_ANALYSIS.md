# Анализ внедрения режима плиток (`btnViewList` / `btnViewTiles`)

## 1) Что было до изменений
- В форме уже существовали кнопки `btnViewList` и `btnViewTiles`, но без обработчиков и логики переключения.
- Таблица заказов (`dgvJobs`) работала как основной и единственный режим отображения.
- Колонка `colPrint` уже формировалась из печатного файла (`stage 3`) при заполнении строк.

## 2) Что реализовано
- Добавлен второй режим отображения: **плитки** на `ListView` с крупными иконками (`View.LargeIcon`).
- Оставлен режим по умолчанию: **таблица** (`List`).
- Реализовано переключение режимов:
  - `btnViewList` -> таблица
  - `btnViewTiles` -> плитки
- Для плиток добавлена фильтрация по условию задачи:
  - показываются только те заказы, у которых есть готовый печатный файл из `colPrint` (фактически существующий файл `stage 3`).
- Добавлена синхронизация выбора:
  - выбор плитки синхронизируется с соответствующей строкой в `dgvJobs`;
  - существующие действия панели (Запустить/Удалить/Папка/Лог и т.д.) продолжают работать без переписывания бизнес-логики.
- Добавлено открытие файла печатного спуска двойным кликом по плитке.
- Счетчик в статус-баре стал режимным:
  - в таблице: `Строк: N | Выделено: M`
  - в плитках: `Плиток: N | Выделено: M`
- Добавлены **реальные PDF-thumbnail**:
  - для `.pdf` рендерится первая страница файла;
  - генерация делается асинхронно в фоне, чтобы интерфейс не подвисал;
  - для не-PDF используется fallback-иконка типа файла.
- Добавлен дисковый кэш thumbnail:
  - кэш хранится в `%LocalAppData%\\MyManager\\ThumbnailCache`;
  - ключ кэша учитывает путь, размер файла, время изменения и размер миниатюры;
  - при повторном открытии миниатюры подхватываются с диска без повторного рендера.
- Подпись плитки переведена на 2 строки (owner-draw):
  - 1 строка: номер заказа (жирный);
  - 2 строка: имя файла печатного спуска.
- Превью подгоняется в квадратный фрейм с сохранением пропорций макета (без растяжения по одной из осей).

## 3) Где изменено
- `Forms/MainForm.cs`

Ключевые добавления:
- Инициализация плиточного вида: `InitializeOrdersTilesView()`
- Инициализация кнопок переключения вида: `InitializeViewModeSwitches()`
- Переключение режимов: `SetOrdersViewMode(...)`
- Наполнение плиток только готовыми файлами печати: `RefreshPrintTilesFromVisibleRows()`
- Синхронизация выбора плитка <-> таблица
- Генерация placeholder-иконок и кэширование иконок по расширению

## 4) Библиотеки и зависимости
### Для тестовой реализации потребовались
- Добавлены NuGet-пакеты для рендера PDF:
  - `PdfiumViewer.Updated` (управляемый API рендера PDF)
  - `PdfiumViewer.Native.x86_64.v8-xfa` (native `pdfium.dll`)
- Плюс штатные возможности WinForms:
  - `System.Windows.Forms` (`ListView`, `ImageList`)
  - `System.Drawing` (отрисовка canvas/масштабирование thumbnail и fallback-иконки)

### Уже существующие в проекте (без изменений)
- `PDFsharp`
- `Svg`
