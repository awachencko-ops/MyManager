# Дизайн таблицы: OLV vs Avalonia (2026-04-02)

## 1. Design Goals
1. Сохранить текущую рабочую логику операторов без сюрпризов.
2. Убрать визуальные фризы при больших списках.
3. Вернуть понятную сортировку с индикатором.
4. Сохранить различимость одиночных и групповых заказов.

## 2. As-is (текущий WinForms стиль)
1. Header + filters + table + status-tray (`Features/Orders/UI/OrdersWorkspace/OrdersWorkspaceForm.Designer.cs`).
2. Плотный кастомный визуал статуса и строк (`Features/Orders/UI/OrdersWorkspace/Core/OrdersWorkspaceForm.StatusCellVisuals.cs`, `.../FileOps/OrdersWorkspaceForm.FileOps.GridInteractions.cs`).
3. Отдельный внешний скролл-контур (`pnlScrollBar`).

## 3. OLV design target
### 3.1 Что сохраняем
1. Текущую палитру и статусные иконки.
2. Разделение `group-order` и `item` визуально.
3. Фильтры в отдельной панели (`flpFilters`), не в заголовке таблицы.
4. Поведение list/tiles переключения.

### 3.2 Что меняем
1. Отрисовку переносим в `FormatRow/FormatCell`.
2. Ветки/иерархию показываем нативно через `TreeListView`.
3. Возвращаем сортировку и sort-indicators.

### 3.3 UX-риски OLV
1. Нужно аккуратно восстановить hover/selected без мерцания.
2. Нужно не потерять stage tooltips и правый клик по колонкам.

## 4. Avalonia design target
### 4.1 Что нравится
1. Чистая tree+table модель (`TreeDataGrid`).
2. Потенциал на более современный UI и масштабируемость.

### 4.2 Почему на скрине было «странно»
1. Без `StyleInclude` для TreeDataGrid контрол в Avalonia не рендерится (пустая область).
2. После добавления theme include это исправлено (`Prototypes/AvaloniaOrdersPrototype/App.axaml:9`).
3. Визуал по умолчанию пока baseline-уровня: нужен дизайн-полиш (плотность, отступы, иконки статусов, конфигурация колонок).

### 4.3 UX-риски Avalonia
1. Придется вручную воспроизвести весь привычный операторам workflow-визуал.
2. Любая недонастройка создаст «чужой» вид интерфейса на этапе внедрения.

## 6. Рекомендация по дизайну
1. Для production migration сейчас: OLV, чтобы сохранить визуальную преемственность и быстро убрать лаги.
2. Для Avalonia: продолжать отдельный UX-прототип и довести до внутреннего дизайн-референса, не ломая текущий рабочий контур.

## 7. Acceptance checklist (для выбранного production контура)
1. Первая колонка показывает `№ заказа` без имен файлов для одиночных заказов.
2. Группы раскрываются/сворачиваются без «прыжков» и потери выделения.
3. Статусные иконки и цвета читаемы при hover/selected.
4. Tooltip ошибок статуса работает.
5. Сортировка и сорт-индикатор работают предсказуемо.
6. Скролл на длинных списках плавный, без фризов.
