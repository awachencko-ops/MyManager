# Parity Checklist OLV vs DGV (2026-04-02)

Цель чеклиста: не переносить таблицу в рабочее окно до полного функционального паритета.

## Блок A. Базовый рендер и дерево
- [x] Отрисовка основных колонок (номер/статус/этапы/даты).
- [x] Group/Item иерархия через `TreeListView`.
- [x] Expand/Collapse (кнопки + клавиатура).
- [x] Статусные иконки в первой колонке (через кастомный renderer без `SmallImageList`, чтобы убрать лишний отступ).
- [x] Сортировка по дате `В препрессе` по умолчанию.

## Блок B. UX и взаимодействие
- [x] Быстрый фильтр с сохранением дерева.
- [x] Сохранение раскрытых групп при обновлении/фильтрации.
- [x] Сохранение выделенной строки при обновлении/фильтрации.
- [x] Контекстное меню строки (копирование номера/имени/статуса/путей).
- [x] Обновление snapshot (`F5` + кнопка `Обновить`).
- [x] Базовый визуальный паритет палитры строк/выделения/ссылочных stage-колонок.
- [x] Иконки статусов и иконки stage-контекстного меню подключены из `Icons/*`.
- [x] Курсор `hand` и tooltip-подсказки для stage-ячеек (путь/не назначен/файл отсутствует).
- [x] Контекстное меню строки усилено (иконки + быстрые действия открыть файл/папку + копирование полей/путей).
- [x] Шапка и плотность строк OLV приведены ближе к боевому стилю (`RowHeight`, `CellPadding`, header style).

## Блок C. File-stage операции
- [x] Клик по стадии `Прием файла` в OLV вызывает open/add поведение как в DGV.
- [x] Клик по стадии `Подготовка` в OLV вызывает open/add поведение как в DGV.
- [x] Клик по стадии `Готов к печати` в OLV вызывает open/add поведение как в DGV.
- [x] Для group-order контейнера блокировка file-stage операций как в DGV.

## Блок D. Архитектура миграции
- [x] Введен каркас `IOrdersGridAdapter`.
- [x] Реализован `DataGridViewOrdersGridAdapter`.
- [x] Добавлен `OlvPrototypeOrdersGridAdapter` (stub для следующего этапа).
- [x] Добавлен feature-flag `REPLICA_USE_OLV_GRID`.
- [x] Интегрирован factory `OrdersGridAdapterFactory`.

## Блок E. Остаток до production-переноса
- [x] Drag&Drop между stage-ячейками в OLV.
- [x] Stage-context-menu в OLV для file-stage (открыть/добавить/удалить/rename/copy-path/paste/open-folder).
- [x] Полная синхронизация `list <-> tiles` на базе адаптера (добавлены write-операции адаптера: применение/очистка выделения в grid).
- [x] Переключение рабочего окна на OLV через feature-flag без прототипной формы (встроенный OLV-контрол в `pnlTable`, DGV оставлен как backend для безопасной совместимости).
- [ ] Smoke-тесты на адаптерный слой + регрессии UI.

## Правило перехода в рабочее окно
Перенос OLV в рабочее окно допускается только после:
1. Закрытия всех пунктов блоков A–E.
2. Green regression/smoke прогона.
3. Отдельного согласования с владельцем продукта.
