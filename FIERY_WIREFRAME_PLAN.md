# Fiery-like wireframe-план для текущего `Form1`

## 1) Цель
Переложить текущие функции `MyManager` на интерфейс, похожий на **Fiery Command WorkStation**, без слома текущей бизнес-логики (`OrderData`, `OrderFileItem`, pipeline и контекстные действия).

---

## 2) Текущая база (что уже есть)
- Верхняя панель действий (`panel1`) и глобальные кнопки режима/лога/настроек.
- Центральный грид `gridOrders` с основными колонками статуса, заказа, стадий и операций.
- Поиск (`txtSearch`) уже размещен над таблицей.
- Группы/элементы (`order` + `item`) и контекстные действия по стадиям.

Это позволяет сделать Fiery-like shell поверх существующих обработчиков.

---

## 3) Wireframe (целевой каркас экрана)

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ TOP BAR: [Создать] [Запустить] [Остановить] [Удалить] [Папка] [Лог] [⚙]     │
├───────────────┬───────────────────────────────────────────┬──────────────────┤
│ LEFT NAV      │ CENTER GRID                                │ RIGHT INSPECTOR  │
│ Очереди       │ Все задания / Фильтр / Поиск               │ Сводка           │
│ - Все         │ ┌───────────────────────────────────────┐   │ Предпросмотр     │
│ - Ожидание    │ │ СОСТОЯНИЕ | № ЗАКАЗА | ...           │   │ Журнал           │
│ - В работе    │ │ ... строки order/item ...            │   │ Ошибки/причины    │
│ - Ошибка      │ └───────────────────────────────────────┘   │                 │
│ - Готово      │                                             │                 │
└───────────────┴───────────────────────────────────────────┴──────────────────┘
```

## 4) Разметка WinForms (конкретно для `Form1`)

### 4.1. Корневой контейнер
- Добавить `SplitContainer` (назовем `splitMain`):
  - `Orientation = Vertical`
  - `Panel1` = LEFT NAV
  - `Panel2` = еще один `SplitContainer` (`splitCenterRight`)

### 4.2. Центр + правая панель
- `splitCenterRight.Orientation = Vertical`
  - `Panel1` = текущая центральная область (`txtSearch`, `gridOrders`, нижний статус)
  - `Panel2` = RIGHT INSPECTOR

### 4.3. Левая панель (LEFT NAV)
- `Panel leftNavPanel`
- `Label "Очереди"`
- `ListBox` или `TreeView` с узлами:
  - Все задания
  - Ожидание
  - В работе
  - Ошибка
  - Готово
  - Частично

### 4.4. Правая панель (RIGHT INSPECTOR)
- `TabControl` с вкладками:
  1. `Сводка`
  2. `Предпросмотр`
  3. `Журнал`
- В `Сводка`:
  - № заказа
  - Текущий статус
  - PitStop/Imposing
  - Пути текущего выделения
- В `Предпросмотр`:
  - placeholder (`Panel + Label`) с текстом “Предпросмотр недоступен”
- В `Журнал`:
  - `TextBox.Multiline` (read-only), наполнение из текущих логов

## 5) Маппинг существующих функций на новый UI

### 5.1. TOP BAR
- `Создать заказ` -> текущий `btnCreateOrder`
- `Запустить` -> текущий `RunForOrderAsync`
- `Лог` -> текущий `OpenLogFile`
- `Настройки` -> текущий `ShowSettingsDialog`

### 5.2. LEFT NAV фильтрация
- Фильтр применяется **до** `FillGrid()` рендера (как сейчас поиск).
- Совмещается с `txtSearch` (AND-логика).

### 5.3. CENTER GRID
- Оставить текущую модель строк `order/item`.
- Для групп:
  - шапка группы = агрегированные поля
  - item-строки = фактические файлы

### 5.4. RIGHT INSPECTOR
- При `SelectionChanged` в `gridOrders`:
  - если order-row -> показать summary order
  - если item-row -> показать summary item (включая `ClientFileLabel` и stage paths)

## 6) Этапы внедрения (безопасно, по шагам)

### Этап A (layout-only)
1. Добавить `SplitContainer`-каркас и новые панели.
2. Переместить текущие контролы центра в `splitCenterRight.Panel1`.
3. Добавить LEFT NAV/RIGHT INSPECTOR как заглушки.

### Этап B (функционал фильтров)
1. Добавить состояние фильтра очереди (`_queueFilter`).
2. Применить в `FillGrid()` вместе с поиском.

### Этап C (инспектор)
1. Подписать `gridOrders.SelectionChanged`.
2. Отображать сводку выделенного order/item.

### Этап D (полировка)
1. Иконки, spacing, шрифты.
2. Горячие клавиши (`Ctrl+F`, `Del`, `F5`).
3. Контекстные подсказки и consistent-палитра статусов.

## 8) Критерии готовности (Definition of Done)
- Экран визуально разделен на **LeftNav / CenterGrid / RightInspector**.
- Фильтр очереди + строка поиска работают совместно.
- Сводка справа обновляется по выделению строки.
- Ни один существующий сценарий (создать/запустить/drag&drop/контекст) не сломан.
