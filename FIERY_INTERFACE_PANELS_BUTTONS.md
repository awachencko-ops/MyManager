# Fiery-like интерфейс: только в рамках текущего функционала MyManager

> Ограничение: список ниже **не добавляет новые модули** (калибратор, ICC-центр, Device Center и т.п.), а опирается только на то, что уже есть в проекте.

## 1) Главный экран (Form1) — обязательные панели

## 1.1 Top Bar (верхняя панель действий)
**Тип:** `Panel + Button`

**Элементы:**
- `MyManager` (заголовок)
- `Settings`
- `Режим: Расширенный / Обычный`
- `Поступление: новые сверху / старые сверху` (переключатель сортировки)
- `Открыть лог`

## 1.2 Панель быстрых действий над списком
**Тип:** `Button + Label + TextBox`

**Элементы:**
- Кнопка `Создать заказ`
- Поле `Поиск`

## 1.3 Центральная таблица заказов
**Тип:** `DataGridView`

**Колонки (как в программе):**
- `СОСТОЯНИЕ`
- `№ ЗАКАЗА`
- `ИСХОДНЫЕ`
- `ПОДГОТОВКА`
- `PITSTOP`
- `HOTIMPOSING`
- `ПЕЧАТЬ`

## 1.4 Нижняя строка состояния
**Тип:** `Label` (status line)

**Назначение:**
- Показывать текущий статус/сообщения обработки.

## 2) Контекстное меню строки заказа (ключевой блок вместо отдельных панелей)

**Тип:** `ContextMenuStrip`

## 2.1 Общие действия
- `🚀 Запустить обработку`

- `↩ Преобразовать группу в одиночный заказ`
- `➕ Добавить строку`
- `❌ Удалить заказ из списка`
- `📁 Открыть папку`

## 2.2 Действия по колонкам

### Для `ИСХОДНЫЕ` / `ПОДГОТОВКА`
- `📋 Вставить путь из буфера`
- `✏️ Переименовать файл`
- `Указать файл...`
- `Удалить файл`

### Для `ПЕЧАТЬ`
- `⏺️ Водяной знак (сверху)`
- `⏺️ Водяной знак (слева)`
- `✏️ Переименовать файл`
- `📋 Копировать путь в буфер`
- `Копировать в Дедушку`
- `Указать файл...`
- `Удалить файл`

### Для `PITSTOP`
- `Открыть диспетчер PitStop`
- `Очистить операцию`

### Для `HOTIMPOSING`
- `Открыть диспетчер Imposing`
- `Очистить операцию`

### Для `СОСТОЯНИЕ`
- `📜 Открыть лог заказа`

## 4) Минимальный Fiery-like набор, который не выходит за рамки программы

Если делаете «визуально как CWS», но без расширения функционала, берите именно это:

1. `Top Bar`
2. `Панель быстрых действий (Создать + Поиск)`
3. `Jobs Grid` (текущие 7 колонок)
4. `Контекстное меню по колонкам`
5. `Строка состояния`

Этого достаточно, чтобы получить интерфейс в стиле рабочей станции оператора, но строго на базе того, что уже поддерживается MyManager.

---

## 5) Что сознательно НЕ включаем

Чтобы не выйти за рамки существующего кода, сейчас не включаем:
- `Device Center` (телеметрия принтера, расходники, лотки)
- `Color Management Center` (ICC-профили, калибровка)
- продвинутый `Preview` с миниатюрами PDF-страниц
- отдельные очереди `Held / Printed / Archived` как в Fiery
- отчеты и аналитические дашборды

Эти блоки можно добавить позже только как отдельные задачи с новой логикой.
