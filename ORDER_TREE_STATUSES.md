# Статусы дерева заказов в MainForm

Файл фиксирует состав узлов в левом дереве заказов (`Forms/MainForm.cs`).

## Пользователь (корневой узел дерева)

- Текущее фиксированное имя для этапа миграции: `Сервер "Таудеми"`.
- Позже будет заменено на выбор пользователя из панели/настроек.

## Состав статусов очереди (treeView1 и cbQueue)

1. `Все задания` — все задания во всех статусах.
2. `Обработанные`
3. `В архиве`
4. `Обрабатывается`
5. `Задержанные`
6. `Завершено`

## Сопоставление рабочих статусов с группами очереди

- `Обработанные`:
  - `Обработано` — файл загружен в программу, структура папок создана.
- `В архиве`:
  - `В архиве` — задание находится в архиве.
- `Обрабатывается`:
  - `Выполняется сборка` — происходит спуск полос в imposing/PitStop.
  - `Обрабатывается` — файл загружается в программу, создается структура папок.
- `Задержанные`:
  - `Отменено` — отмена в процессе сборки спуска.
  - `Ошибка` — завершено неуспешно, ошибка.
- `Завершено`:
  - `Завершено` — успешно завершено.

## Синхронизация пользователя и списка очереди

- Пользователь выбирается в `treeView1` (корневой узел слева).
- `cbQueue` в шапке показывает статусы выбранного пользователя:
  - `Все задания`
  - `Обработанные`
  - `В архиве`
  - `Обрабатывается`
  - `Задержанные`
  - `Завершено`
- Выбор статуса в `cbQueue` выбирает соответствующий дочерний узел статуса в `treeView1` у текущего пользователя.
- При выборе узла статуса в `treeView1` значение `cbQueue` синхронизируется с этим статусом.
